

## 1. 서비스 개요 및 아키텍처

본 서비스는 사용자가 웹서핑 중 수집한 데이터를 자동으로 분석·분류하여 개인 지식 베이스(Second Brain)를 구축하는 시스템입니다.

### 💡 데이터 흐름 (Data Flow)

1. **Chrome Extension:** 웹페이지에서 드래그한 텍스트 또는 전체 URL 수집 및 Supabase API로 전송.
    
2. **Supabase (Backend/DB):** 수집된 본문 데이터 저장 및 Edge Functions를 활용한 형태소 분석(단어 빈도, 키워드 추출, 연관성 계산).
    
3. **Obsidian Plugin:** Supabase와 실저장소(Local Vault) 간 양방향 동기화. Webhook 또는 폴링 방식을 통해 로컬에 `.md` 파일 생성.
    
4. **Web App (Vite + Vercel):** Supabase에 적재된 데이터를 기반으로 연결성 그래프, 단어 클라우드, 카테고리별 분류 대시보드 제공.


## 2. 핵심 기능 명세 (PRD)

### 2.1 크롬 익스텐션 (Chrome Extension)

- **컨텍스트 메뉴 클리핑:** 웹페이지에서 드래그 후 우클릭 메뉴를 통해 특정 텍스트만 수집.
    
- **원클릭 페이지 저장:** 상단 익스텐션 아이콘 클릭 시 현재 페이지의 Title, URL, Meta Description 수집.
    
- **태그 및 메모 추가:** 클리핑 시점에 사용자가 직접 간단한 태그나 한 줄 메모를 추가할 수 있는 팝업 UI.


### 2.2 옵시디언 플러그인 (Obsidian Plugin)

- **Supabase 실시간 동기화:** Supabase DB의 신규 클리핑 데이터를 감지하여 지정된 Vault 내에 Markdown 파일(`YYYY-MM-DD-제목.md`) 생성.
    
- **메타데이터(Frontmatter) 자동 삽입:** 파일 상단에 원본 URL, 수집 일시, 추출된 핵심 키워드 및 연관 단어를 Frontmatter 형식으로 자동 기록.
    
- **로컬 수정사항 백업 (선택):** 옵시디언 내에서 수정된 내용이 있을 경우 Supabase로 역동기화.
    

### 2.3 웹 대시보드 (Vite + Web App)

- **연결성/관련성 그래프 시각화:** 단어 빈도 및 키워드 유사도를 기반으로 노트 간의 관계를 2D/3D 그래프(예: Force-directed graph)로 시각화.
    
- **통계 리포트:** 주로 디깅한 분야의 키워드 빈도수 통계, 타임라인 뷰 제공.
    

## 3. 기술 스택 및 데이터베이스 구조

### 🛠 Tech Stack

- **Frontend:** Vite, React (or Vue/Svelte), TypeScript, TailwindCSS
    
- **Deployment:** Vercel (Web App)
    
- **Backend & DB:** Supabase (PostgreSQL, Edge Functions, Realtime)
    
- **Extensions:** Manifest V3 (Chrome), Obsidian Plugin API
    

### 🗄 Database Schema (Supabase)

SQL

```
-- 1. 유저 테이블 (기본 auth.users 연동)
create table profiles (
  id uuid references auth.users on delete cascade primary key,
  updated_at timestamp with time zone,
  username text unique
);

-- 2. 클리핑 데이터 테이블
create table clippings (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references profiles(id) on delete cascade not null,
  title text not null,
  url text,
  content text,              -- 드래그한 본문 또는 요약
  raw_html text,             -- 필요시 원본 구조 저장
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 3. 단어 빈도 및 키워드 분석 결과 테이블
create table analysis_results (
  id uuid default gen_random_uuid() primary key,
  clipping_id uuid references clippings(id) on delete cascade not null,
  keywords jsonb,            -- { "단어1": 5, "단어2": 3 } 형태로 빈도수 저장
  category text,             -- 분류 알고리즘 기반 자동 카테고리
  related_clipping_ids uuid[] -- 연관성 있는 다른 클리핑 ID 배열
);
```

## 4. 로직 설계: 단어 빈도 및 연관성 처리 방법

가지고 계신 자료를 적용할 핵심 백엔드 로직 프로세스입니다. Supabase Edge Functions(Deno 환경)를 활용하여 서버리스로 처리하는 방식이 가장 깔끔합니다.

```
[웹 클리핑 발생] 
       ↓
[Supabase DB 저장] 
       ↓
[Edge Function 트리거] 
       ↓
[텍스트 전처리 & 형태소 분석 (명사 추출)]
       ↓
[TF-IDF 또는 단어 빈도수(Count) 계산]
       ↓
[기존 데이터와 자카드 유사도 / 코사인 유사도 비교]
       ↓
[연관성 테이블(analysis_results) 업데이트]
```

- **옵시디언 연동:** 분석이 완료되면 Supabase Realtime 또는 옵시디언 플러그인 전용 API 엔드포인트를 통해 로컬 플러그인이 변경 사항을 감지하고, 연관 단어를 `[[링크]]` 형태로 변환하거나 Frontmatter에 주입하여 로컬에 저장합니다.
